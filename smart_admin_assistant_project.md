# المساعد الإداري الذكي — كلية الحاسبات والذكاء الاصطناعي
## Smart Administrative Assistant — Faculty of Computers & AI, Al-Riyada University

> **نوع المشروع:** AI Agent محلي يعتمد على RAG  
> **القيود التقنية:** أدوات مجانية ومفتوحة المصدر بالكامل — بدون اشتراكات مدفوعة  
> **مصدر البيانات الحالي:** الموقع الرسمي للكلية فقط (لا يوجد وصول لملفات داخلية)  
> **إجمالي مدة التنفيذ:** 12 أسبوع

---

## 1. فكرة المشروع الأساسية

بناء وكيل ذكاء اصطناعي (AI Agent) يعمل محلياً على أجهزة الكلية، مخصص لدعم موظفي الشؤون الإدارية والطلابية. يعتمد النظام كلياً على بيانات الكلية الرسمية المتاحة للإجابة الفورية والدقيقة على الاستفسارات الإدارية المتكررة، مما يقلل العبء على الموظفين ويسرّع تقديم الخدمة للطلاب.

## 2. المكدس التقني الكامل (Tech Stack)

| المكوّن | الأداة المختارة | السبب |
|---|---|---|
| نموذج اللغة (محلي) | Ollama + Mistral-7B أو Gemma-2-9B | يعمل بالكامل offline — بدون API key |
| نموذج اللغة (احتياطي) | Groq Free API — LLaMA-3-8B | 14,400 طلب/يوم مجاناً |

## 4. المكوّنات الجديدة الجوهرية

### 4.1 بوابة الثقة (Confidence Gate)

```python
# مثال مبسط على منطق بوابة الثقة
CONFIDENCE_THRESHOLD = 0.38

def confidence_gate(query: str, retriever) -> dict:
    results = retriever.get_relevant_documents_with_scores(query)
    
    if not results:
        return {"action": "fallback", "reason": "no_results"}
    
    best_score = max(score for _, score in results)
    
    if best_score >= CONFIDENCE_THRESHOLD:
        return {
            "action": "send_to_llm",
            "chunks": [doc for doc, _ in results],
            "score": best_score
        }
    else:
        # سجّل في Gap Logger
        log_gap(query, best_score)
        return {
            "action": "fallback",
            "reason": "low_confidence",
            "score": best_score
        }
```

**لماذا هذا المكوّن حيوي؟**  
يمنع النموذج من "اختراع" إجابات عندما لا توجد بيانات كافية. العتبة `0.38` تُضبط خلال Sprint 2 بناءً على التجارب الفعلية.

### 4.2 Gap Logger (مسجّل الثغرات)

```python
import json
from datetime import datetime
from pathlib import Path

GAP_LOG_PATH = Path("/logs/query_gaps.jsonl")

def log_gap(query: str, best_score: float, language: str):
    entry = {
        "timestamp": datetime.now().isoformat(),
        "query": query,
        "best_similarity_score": round(best_score, 4),
        "detected_language": language
    }
    with open(GAP_LOG_PATH, "a", encoding="utf-8") as f:
        f.write(json.dumps(entry, ensure_ascii=False) + "\n")
```

**كيف يُستخدم Gap Log استراتيجياً؟**

1. بعد كل UAT session، افتح `query_gaps.jsonl`
2. صنّف الأسئلة المتكررة في مجموعات (تسجيل، رسوم، إجازات...)
3. استخدم هذا التقرير كـ **دليل رسمي** لطلب الوثائق الداخلية من إدارة الجامعة
4. أجب على الأسئلة القابلة للإجابة يدوياً وأضفها إلى `faq_manual.md`

### 4.3 منسّق الإجابة المتدرج (Tiered Response Formatter)

| مستوى الثقة | نوع الإجابة | مثال |
|---|---|---|
| **عالية** (≥ 0.38) | إجابة كاملة + مصدر | "الحد الأقصى للغياب 25% وفقاً للائحة الدراسية، المادة 14 — [موقع الكلية]" |
| **متوسطة** (0.25–0.37) | إجابة جزئية + تحفظ | "بناءً على الموقع، يبدو أن... لكن يُنصح بالتأكيد من مكتب شؤون الطلاب" |
| **منخفضة** (< 0.25) | إحالة مباشرة | "هذه المعلومة غير متوفرة حالياً. تواصل مع: قسم شؤون الطلاب — داخلي 123" |

### Sprint 3 — نواة RAG + بوابة الثقة + منطق الـ Fallback
**المدة:** الأسبوعان 6–7

| المهمة | التفاصيل |
|---|---|
| بناء RetrievalQA Chain | استعلام → استرجاع من 3 مجموعات بالتوازي → دمج النتائج → أعلى 5 chunks |
| تطبيق Confidence Gate | فحص `max(similarity_scores)` قبل إرسال أي شيء للـ LLM |
| تطبيق Gap Logger | كتابة كل استعلام منخفض الثقة في `query_gaps.jsonl` |
| 3 قوالب Prompt | عالي الثقة / متوسط الثقة / لا توجد بيانات |
| Fallback ذكي | لا تقل أبداً "لا أعرف" — دائماً أضف جهة التواصل المناسبة |
| ذاكرة المحادثة | `ConversationBufferWindowMemory` آخر 4 أدوار |
| توجيه اللغة | كشف AR/EN تلقائياً وتوجيه للقالب المناسب |

**✓ التسليم:** خط RAG كامل يعمل + اختبار 20 سؤالاً بنتائج موثقة

## 10. مؤشرات الأداء الرئيسية (KPIs)

| المؤشر | الهدف | كيفية القياس |
|---|---|---|
| Hit Rate @ k=5 | ≥ 75% | هل الـ chunk الصحيح ضمن أعلى 5 نتائج؟ |
| Faithfulness | ≥ 80% | هل الإجابة تطابق مصدرها؟ |
| Fallback Precision | ≥ 90% | هل الأسئلة المنخفضة الثقة أُحيلت بشكل صحيح؟ |
| تغطية الموقع | ≥ 95% من الصفحات | عدد الصفحات المُستخرجة / إجمالي صفحات الكلية |
| رضا الموظفين (UAT) | ≥ 4/5 | متوسط تقييم الموظفين بعد الاستخدام الفعلي |

## 11. التثبيت والتشغيل السريع

```bash
# 1. استنساخ المشروع
git clone https://github.com/your-repo/smart-admin-assistant.git
cd smart-admin-assistant

# 2. إنشاء البيئة الافتراضية
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 3. تثبيت المتطلبات
pip install -r requirements.txt

# 4. إعداد متغيرات البيئة
cp .env.example .env
# عدّل .env وأضف GROQ_API_KEY إذا أردت الاستخدام السحابي

# 5. تشغيل Ollama وسحب النموذج
ollama pull mistral

# 6. تشغيل زحف الموقع
python ingestion/crawl.py

# 7. استيعاب البيانات في ChromaDB
python ingestion/ingest.py

# 8. تشغيل التطبيق
bash run.sh
# افتح المتصفح على: http://localhost:7860
```
